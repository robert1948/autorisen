# UI & UX Playbook

- **Goal:** Maintain a consistent product experience by pairing product requirements with up-to-date Figma sources of truth.
- **Scope:** UX research synthesis, UI component updates, accessibility review, and design handoff for engineering.

## Cadence

- Weekly design critique with engineering representation.
- Monthly accessibility audit covering new UI surfaces.
- Quarterly component library refresh in Figma and Storybook.

## Checklist

- Validate problem statement with at least 3 user interviews or support transcripts.
- Update relevant Figma flows and sync frame links in issue tracker.
- Run component contrast check (WCAG AA minimum).
- Attach redlines and motion specs before handoff.
- Confirm engineers acknowledge design handoff and edge cases.

## Tooling

- Figma for source files and prototypes.
- FigJam for collaborative workshops.
- Storybook (planned) for component verification.
- Loom for asynchronous design walkthroughs.

## Owners

- **Primary:** Product Design lead.
- **Support:** Frontend lead, Accessibility SME, Product Manager.
